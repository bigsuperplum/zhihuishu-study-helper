# 智慧树作业作答脚本开发提示词（专业增强版）

请你扮演一位资深的前端开发工程师和 Tampermonkey（油猴）脚本编写专家。你需要编写一个**高可用、高鲁棒性**的油猴脚本，实现在「智慧树（知到）官网」作答作业的功能。

## 1. 业务流程与场景描述
目标网站为智慧树官网（https://www.zhihuishu.com/）及相关的学习页面前端路由。
标准的业务路径如下：
1. 登录后，从主页或“我的学堂”进入“智慧课堂”。
2. 进入具体课程页面，点击“任务·作业·考试”栏目，然后进入“作业”子版块。
3. 脚本需自动扫描当前页面所有状态为“未完成”的作业，并点击“去完成”或“继续答题”进入作答界面。
4. **核心作答逻辑**：在作答界面完整提取题目文本和选项（包括单选题、多选题、判断题），将提取到的内容组装为 Prompt，请求本地或远程配置的大语言模型（LLM），获取答案后显示在网页上。

## 2. 脚本开发核心要求（重点关注鲁棒性与防检测）

### 2.1 脚本元数据与权限要求
- `@name`: 智慧树AI作业全自动作答助手
- `@match`: `*://*.zhihuishu.com/*` （请根据实际路由优化匹配规则，确保在主页、作业列表和作答页均能注入）。
- `@grant`:
  - `GM_xmlhttpRequest`: 必须项。用于绕过跨域限制，向本地或云端大模型 API 发起网络请求。
  - `GM_setValue` / `GM_getValue`: 必须项。用于存储大语言模型的配置项（API URL、API Key、Model Name 等）。

### 2.2 动态 DOM 与 SPA 适配（重要！）
智慧树页面是典型的单页应用（SPA）或具有动态渲染特性的页面：
- **禁止使用死等**：严禁使用固定的 `setTimeout` 进行元素查找。你必须实现一个健壮的 `waitForElement(selector, timeout)` 函数，内部基于 `MutationObserver` 或 `setInterval` 轮询，直到元素出现再进行后续操作。
- **SPA 路由监听**：脚本需要能识别页面路由（URL fragment）的变化，避免在页面内容通过 AJAX 更新时脚本失效。
- **元素兼容**：脚本应具备良好的 DOM 解析能力，灵活提取题干和选项，并考虑单选、多选、判断题可能由于页面改版产生的 DOM 结构差异。

### 2.3 大语言模型（LLM）通信与解析
- **配置化面板**：脚本应在页面边缘注入一个简单的浮动配置面板（或通过油猴菜单），允许用户自行填写及修改 `Base URL`、`API Key` 和 `Model Name`（默认支持与 OpenAI API 格式兼容的模型，如本地基于 Codex/Ollama 部署的模型）。
- **结构化输出约束**：在与大模型通信的 `System Prompt` 中，必须严格要求大模型**仅返回 JSON 格式字符串**，例如 `{"answer": ["A", "C"], "reason": "xxx"}`。**严禁大模型输出多余的自然语言文本**，以免前端代码解析失败（需在捕获结果时加入 `JSON.parse` 及 try-catch 和重试机制）。
- **异常重试**：当大模型接口超时或返回结果无法解析时，最多进行 3 次重试。若依然失败，则在页面上给予显著提示。

## 3. 产出与验收标准
1. 输出完整的、可直接在浏览器中安装运行的油猴脚本代码（包含详细中文注释）。
2. 代码必须结构清晰且模块化（DOM 选择与监听模块、网络请求模块、UI 控制模块等结构独立）。
3. 必须提供一份简要的 readme，说明如何配置 AI 参数（特别是配置本地 Codex 模型）以及后续 DOM 变更时用户应如何调整选择器。

请你综合考虑上述所有要求，给出实现该油猴脚本的完整代码。
